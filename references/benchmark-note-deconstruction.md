# 同行与爆款笔记拆解表

这个文档用于处理用户问“帮我拆同行”“这个爆款为什么爆”“这篇笔记能不能学”。

核心原则：拆爆款不是抄整篇，而是拆可迁移元素、不可迁移变量和风险边界。

---

## 1. 先判断能不能学

看到爆款后，先问：
- 它吸引的人群是不是我要的人群
- 它卖的产品或服务是否和我相近
- 它的价格带、信任门槛、承接方式是否适配我
- 它的数据是否可能来自达人身份、投放、平台活动或账号历史积累
- 它有没有合规风险或低创风险

如果人群、业务、承接都不适配，只能参考局部元素。

---

## 2. 十维拆解表

```markdown
| 维度 | 要看什么 | 结论 |
|---|---|---|
| 卖什么 | 产品、服务、身份感、结果、解决方案 |  |
| 写给谁 | 明确人群、隐含人群、排除人群 |  |
| 需求 | 痛点、痒点、决策问题、情绪触发 |  |
| 搜索词 | 标题、正文、话题、图片文字里的关键词 |  |
| 标题钩子 | 情绪、结果、反差、避坑、清单、提问 |  |
| 封面/首屏 | 画面、文字、人物、场景、视觉焦点 |  |
| 正文结构 | 开头、证据、信息密度、节奏、CTA |  |
| 信任证据 | 真实经历、案例、数据、细节、专业判断 |  |
| 评论问题 | 用户追问、质疑、购买前顾虑 |  |
| 承接入口 | 评论、私信、店铺、合集、群、表单等 |  |
```

---

## 3. 可复用元素

优先复用这些：
- 选题角度
- 用户问题
- 标题情绪
- 封面构图思路
- 首屏表达顺序
- 证据类型
- 评论区二次解释方式
- 内容节奏
- 搜索词组合

不要直接复用这些：
- 原文句子
- 原图构图细节
- 虚假人设
- 私域导流话术
- 夸张承诺
- 不适配自己的价格带和客群

---

## 4. 爆款元素拆解

拆爆款时，默认输出：

```markdown
## 爆款元素
- 爆的是选题：
- 爆的是标题：
- 爆的是封面：
- 爆的是人群情绪：
- 爆的是信任证据：
- 爆的是评论区问题：
- 可迁移元素：
- 不可迁移变量：
- 合规风险：
```

---

## 5. 小号和腰部账号更值得看

不要只盯头部账号。

小号或腰部账号里突然跑出来的数据，往往更能说明：
- 选题本身有需求
- 画面或标题有截停力
- 某个搜索词正在被用户使用
- 用户评论区有未被满足的问题

判断数据时，要看它在该账号内部是否异常好，而不是只看绝对点赞数。

---

## 6. 拆完后的动作

每次拆完，不只给分析，还要给下一步：
- 复用一个选题角度
- 换一个人群切口
- 换一个场景
- 换一组关键词
- 换一种证据
- 保留结构但重写观点
- 用评论区问题做下一篇

---

## 7. 输出模板

```markdown
## 同行笔记拆解
- 这篇主要卖的是：
- 它吸引的人群：
- 用户需求：
- 流量入口：搜索 / 瀑布流 / 两者兼顾
- 最强爆点：
- 可学习的 3 个元素：
- 不能照搬的 3 个点：
- 适合你改成的选题：
- 下一篇建议：
```
