# 合规边界

这个文档用于处理小红书获客、评论区、私信、达人合作、素人号、员工号、店铺和群承接中的风险边界。

核心判断：课程里的实操经验只能提炼为调研、内容、复盘和承接原则，不能转写成绕审核、规避平台规则或虚假营销的执行方案。

---

## 1. 不提供的内容

遇到以下请求，只做风险提示和合规替代：

- 绕过平台审核
- 暗号、谐音、拆字规避识别
- 诱导站外导流
- 批量私信骚扰
- 虚假达人、虚假素人、虚假员工号
- 虚假测评、虚假评论、伪造用户反馈
- 夸大疗效、收益、效果、成功率
- 批量搬运、洗稿、低质换皮
- 恶意混入同行群、骚扰或破坏他人社群

---

## 2. 可以提供的安全替代

可以建议：

- 平台允许的站内承接方式
- 店铺、团购、表单、官方组件
- 评论区公开答疑
- 私信中的合规信息说明
- 合集、置顶笔记、账号简介做信息整理
- 用内容筛选客户，而不是用话术诱导客户
- 达人合作前明确评论区维护和事实边界

---

## 3. 评论区处理原则

评论区是第二内容现场，但不是灰色转化场。

可以做：
- 回答真实问题
- 补充笔记未写完的信息
- 提炼下一篇选题
- 用置顶评论做公开说明
- 对不适合的用户做预期管理

不要做：
- 明示或暗示绕平台导流
- 用暗号诱导用户
- 制造虚假热度
- 删除正常质疑只保留好评

---

## 4. 私信处理原则

私信承接要更像咨询筛选，而不是强推成交。

推荐顺序：

```text
确认需求 -> 补充信息 -> 判断适配 -> 给出下一步
```

避免：

```text
一上来索要联系方式 -> 夸大承诺 -> 制造焦虑 -> 强行成交
```

---

## 5. 输出要求

当用户问承接、导流、私信、评论区、达人号、素人号、员工号时，默认输出：

```markdown
## 风险判断
- 风险等级：低 / 中 / 高
- 风险点：
- 不建议做：

## 合规替代
- 站内承接：
- 评论区处理：
- 私信表达：
- 内容补强：
```

如果用户明确要求违规执行步骤，拒绝执行步骤，只给安全替代。
