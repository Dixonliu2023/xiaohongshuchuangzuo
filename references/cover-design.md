# 封面建议参考

这个文档服务于主 Skill 的“封面建议”模块。

封面的核心不是“好看”，而是：
- 一眼看懂
- 一眼感知价值
- 一眼知道要不要点

---

## 一、先判断封面类型

| 类型 | 适合内容 | 特点 |
|---|---|---|
| 大字报 | 痛点、观点、教程 | 文字驱动，截停强 |
| 清单图 | 合集、步骤、模板 | 信息量强，适合收藏 |
| 对比图 | 改造、结果、案例 | 价值感知强 |
| 场景图 | 体验、产品、生活方式 | 氛围感强 |
| 人物图 | 人设、故事、经验 | 信任感强 |

---

## 二、封面四要素

### 1. 主文案

要求：
- 只讲一个重点
- 最好 6-14 字
- 尽量让人一眼看懂

---

### 2. 副文案

作用：
- 补充结果、对象或范围

不要：
- 和主文案重复
- 写成一整段

---

### 3. 视觉焦点

必须让用户一眼知道先看哪里。

常见焦点：
- 大标题
- 对比主体
- 人物表情
- 关键数字

---

### 4. 版式

原则：
- 一屏只服务一个重点
- 不要把所有信息都放首图
- 留白比堆满更有用

---

## 三、输出模板

```markdown
# 封面建议

## 封面类型
- 大字报 / 清单图 / 对比图 / 场景图 / 人物图

## 主文案
...

## 副文案
...

## 视觉重点
- ...

## 配色方向
- ...

## 版式建议
- ...
```

---

## 四、常见错误

- 错误 1：主标题太长，看不清重点
- 错误 2：副标题重复主标题
- 错误 3：设计感强，但价值感知弱
- 错误 4：把正文内容都塞在封面里
