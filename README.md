# 小红书创作助手

一个面向中文场景的小红书创作决策与诊断 skill。

它不是“万能代写器”，也不是单纯的标题生成器。  
它更适合用来解决这些问题：

- 我这个账号到底该怎么定位
- 我这个产品在小红书上该怎么讲
- 这篇笔记到底写给谁
- 为什么这篇内容发出去没有咨询
- 标题、封面、首屏、正文，到底是哪一环出了问题

尤其适合：
- 旅行社
- 定制游
- 包车游
- 路线产品
- 徒步产品
- 想把卖点写成“用户能感知到的价值”的创作者

---

## 这个 skill 能做什么

### 1. 账号定位

当你只有一个模糊方向，或者产品很多、表达很散时，
它会先帮你判断：

- 你到底在服务谁
- 你卖的到底是什么
- 你的账号主轴该怎么收拢

### 2. 任务路由

它不会默认所有请求都是“帮我写篇小红书”。

它会先判断你当前更需要的是：
- 定位
- 选题
- 标题
- 封面
- 爆款拆解
- 内容诊断
- 合规检查
- 直接生成成稿

### 3. 标题与封面

它会优先处理：
- 标题是否有对象感、结果感、点击动机
- 封面是否能一眼传达价值
- 首屏是否能让用户停下来

内置规则包括：
- 小红书主标题不能超过 20 个字，且标点也计入
- 封面要和标题、首屏前 3 句一起判断
- 旅行收客内容优先传达“决策价值”，不是只放风景

### 4. 内容诊断与重写

如果你已经有草稿、截图、链接、爆款样本，
它优先做诊断，而不是一上来重写。

会重点判断：
- 这篇内容在卖什么
- 写给谁
- 为什么这个人现在会点
- 为什么这个人会来问你
- 有没有“像资料，不像人在说话”的问题

### 5. 旅行与路线产品表达

这是这个 skill 的一个重点场景。

它特别适合处理：
- 大朝台
- 城市体验产品
- 包车定制
- 亲子线路
- 跨城拼图路线

处理这类内容时，它默认优先回答：
- 这篇写给谁
- 这个人现在为什么会来
- 为什么来这里
- 为什么要选你
- 他会担心什么
- 你这条线到底解决了什么现实问题

### 6. 合规与边界

它不会为了转化，去给高风险表达开口子。

默认会规避：
- 夸大承诺
- 擦边导流
- 灰色私域导流
- 虚假宣传
- 过度营销化表达

---

## 它和普通“小红书写作工具”有什么不同

这个 skill 的核心不是“写得像”，而是“判断得准”。

它更强调这几件事：

- 先判断任务，再决定怎么写
- 先做人群和业务判断，再谈爆款和文案
- 先看用户真实反馈，再校准内容方向
- 先验证标题、封面、首屏，再放大投入
- 平视表达比资料堆砌更适合当下平台
- 公信力不能过早被销售意图污染

一句话概括：

> 这不是一个帮你批量生产文案的 skill，  
> 而是一个帮你把小红书内容做得更接近真实获客链路的 skill。

## 特别适合谁

如果你符合下面任意一种，这个 skill 会比较有用：

- 你有产品，但不会写成小红书内容
- 你会写文案，但发出去没人问
- 你总觉得内容“没灵魂”
- 你能拆爆款，但一到自己写就不转化
- 你是旅行行业，想把路线讲得更像“对的人会心动的内容”
- 你不想做低创换皮，想沉淀一套长期可复用的方法

---

## 当前这套 skill 的核心方向

这个 skill 当前默认服务的一个重要 IP 主轴是：

`懂高压人群状态切换的人`

所以在处理旅行、徒步、城市体验、路线产品时，
它不只是写景点和卖点，
而是更关注：

- 用户现在处于什么状态
- 为什么他此刻需要这趟旅行
- 这趟体验能帮他切换什么
- 为什么你是更适合带他完成这次切换的人

这套思路目前已经深度应用在：
- 大朝台徒步产品表达
- 成都状态切换型产品表达
- 定制游/包车产品表达

## 适用边界

这个 skill 不适合做这些事：

- 单纯追求批量洗稿
- 做灰色导流
- 写违规承诺
- 用“像爆款”代替“有判断”
- 用“会写文案”代替“懂业务”

它更适合长期主义的做法：
- 边做边校准
- 边写边诊断
- 边输出边沉淀方法
